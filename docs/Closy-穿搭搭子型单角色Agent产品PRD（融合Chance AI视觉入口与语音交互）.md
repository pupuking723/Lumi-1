# Closy：穿搭搭子型单角色 Agent 产品 PRD（融合 Chance AI 视觉入口与语音交互）

## 面向未来 Agent Native 社交平台的前期切入口产品

---

## 1. 产品概述

### 1.1 产品名称（暂定）
代号：**Closy**

可替代命名方向：
- Closy
- Coco
- OOTD Buddy
- Stylemate
- 小搭子

此处先统一用 **Closy**。

### 1.2 一句话定义

**Closy 是一个以穿搭 / OOTD / 审美决策为第一入口，但核心在于陪伴用户进行自我表达、状态整理、轻关系应对和日常小决策的单角色 Agent。**

它不是单纯的穿搭建议工具，也不是泛化的陪聊机器人，而是一个：

- 懂你
- 记住你
- 陪你
- 有明确个性边界
- 能在特定生活语境中长期存在

的关系型角色 Agent。

同时，Closy 会明确吸收 Chance AI 的一个关键启发：

> **让视觉入口前置，让“看到 / 拍到 / 立即问”成为比“先想清楚再打字”更自然的使用起点。**

因此，Closy 不只是一个文字聊天产品，而是一个：

- **可直接自拍**
- **可即时拍摄**
- **可上传 look / 商品 / 截图**
- **可文字聊**
- **可语音聊**

的多模态关系型 Agent。

### 1.3 产品本质

表层上，Closy 帮用户解决的是：

- 今天这套行不行
- 哪套更适合我
- 这件值不值得买
- 我适合什么风格
- 我今天应该呈现成什么感觉

但更深层上，Closy 解决的是：

- 我希望有人懂我
- 我希望有人陪我一起做微小但频繁的决定
- 我希望在自我表达和社交呈现上被理解
- 我希望有一个对象能持续记得我、越来越懂我
- 我希望它不只是回答问题，而是陪我生活里的一部分

因此，**穿搭是入口，关系是本体；功能是抓手，陪伴是留存核心。**

进一步说，Closy 的产品姿态不是“文本框里加一个图片能力”，而是：

> **让视觉输入、自拍、即时拍摄、语音表达与文字对话共同成为关系入口。**

## 2. 产品背景与战略位置

### 2.1 长期目标：Agent Native 社交平台

我们的长期目标，不是做一个穿搭工具 App，也不是做一个单点角色聊天产品，而是构建一个 **Agent Native 社交平台**：

- 人与人共存
- 人与角色 Agent 共存
- 多角色 Agent 共存
- 会话、关系、内容、互动和服务都围绕 Agent-native 结构展开

在这个终局里，平台会包含很多不同类型的角色 Agent，例如：

- 穿搭搭子
- 社交军师
- 生活顾问
- 行动教练
- 情绪陪伴角色
- 消费决策角色
- 各种创作者 / IP / 品牌角色

这些角色最终将共同组成一个多角色、多关系、多内容、多互动的中心化 Agent Native 社交前台。

### 2.2 为什么前期不直接做多角色平台

虽然长期目标是多角色平台，但前期不应一上来就做：

- 多角色系统
- 群聊 / 房间
- 完整社交关系网络
- 内容社区 feed
- 多层供给和复杂分发

原因有三：

#### 1）前期最需要验证的是“关系能否成立”
相比复杂网络，前期最核心的问题是：

- 用户会不会把一个角色当作固定对象反复找
- 用户会不会因为“它懂我”而形成关系感
- 用户会不会在任务之外也回来找它
- 用户会不会对一个角色产生高频打开习惯

这些问题更适合用单角色产品验证。

#### 2）过早做多角色会模糊验证目标
如果一开始就上多角色、社交网络、房间、内容 feed，会同时叠加很多变量：

- 角色成立与否
- 内容成立与否
- 分发成立与否
- 社交活跃成立与否
- 冷启动成立与否

这样会看不清到底什么成立、什么没成立。

#### 3）单角色先成立，是长向多角色平台的合理路径
前期最好的起点，是先让**一个关系型角色**成立，再逐步外扩到：

- 更多角色
- 更多场景
- 更多内容
- 更多互动
- 更多社交结构

所以，当前这个单角色产品，不是终局缩小版，而是**终局的验证入口版**。

## 3. 核心产品方法论

### 3.1 关键结论一：陪伴感是共性，角色价值是个性

这是整个产品设计的底层方法论。

#### 为什么要这么定义？

未来多角色 Agent 平台里，几乎所有角色都会具备：

- 懂你
- 记住你
- 回应你
- 关心你
- 延续关系
- 提供某种陪伴感

这些能力未来会成为平台基础设施，就像通知、搜索、推荐一样重要，但它们**不能构成角色本身的差异化**。

如果一个角色的定义只停留在：

- 我会陪你
- 我懂你
- 我会记得你

那么未来所有角色都会变成：

> **不同皮肤、不同口吻、但本质相似的万能陪伴体。**

这会导致严重同质化。

因此必须区分：

#### 共性层：平台级陪伴能力
所有角色都可以有：
- 长期记忆
- 情绪承接
- 关系延续
- 主动消息
- 陪伴式对话

#### 个性层：角色独特价值
每个角色必须回答：
- 它在哪些时刻最值得被找？
- 它从什么角度理解用户？
- 它通过什么判断体系陪用户？
- 它在用户生活中处理哪一类独特问题？
- 它的陪伴“形状”是什么？

所以：

> **陪伴感是平台的共性，角色价值才是角色的个性。**

### 3.2 关键结论二：不是更通用，而是更完整

前面一个很重要的误区是：如果角色不能只停留在功能任务里，那是不是要做得“更通用”？

答案是：**不能这么表达。**

#### 为什么“更通用”是危险的？

因为“更通用”很容易滑向：
- 什么都能聊
- 什么都能接
- 什么都能陪
- 没有边界
- 没有明确视角
- 谁都能替代

这最终会把角色做成：
**泛化陪聊壳 + 一层功能外衣**。

#### 正确方向是什么？

不是“更通用”，而是：

> **在不丢掉角色个性边界的前提下，让角色在功能场景之外，也拥有更丰富的非任务型陪伴场景。**

也就是说：
- 不是去掉角色边界
- 而是让角色不只活在功能任务里

所以拿穿搭搭子来说：

错误表达：
- 它不能只聊穿搭，应该更通用一些

正确表达：
- 它不能只活在穿搭任务里，应该在**审美、自我表达、社交呈现、状态感**这些与它强相关的语境里，拥有更完整的陪伴场景

因此：

> **不是更通用，而是更完整。**

### 3.3 关键结论三：功能是入口，角色不能只活在功能里

任何角色如果只在任务发生时才被找，就更像工具，而不是关系对象。

真正有潜力长成 Agent Native 社交角色的产品，必须同时具备：

#### 1）明确的功能入口
给用户第一次来、反复来、站外传播一个清晰理由。

#### 2）与角色价值强绑定的非任务陪伴
让用户在没有明确任务时，也会因为这个角色的独特视角而回来。

对 Closy 来说：
- 功能入口：穿搭 / ootd / 买前决策 / 二选一
- 非任务陪伴：围绕风格认同、自我表达、社交呈现、状态感展开

### 3.4 关键结论四：吸收 Chance AI 的不是“视觉产品形态”，而是“视觉优先的入口方法”

Closy 不应该原样模仿 Chance AI 成为“纯视觉 Agent”，但应明确吸收它最有价值的一层：

> **把视觉入口前置，让视觉成为用户更自然的第一触点。**

这意味着：
- 用户不一定先打字
- 用户可以先自拍
- 用户可以先拍 look
- 用户可以先拍商品
- 用户可以先发 story 截图、购物截图、镜子照
- Agent 先理解视觉对象，再进入判断、解释、陪伴和对话

因此，Closy 的核心界面不应只是一个文本输入框，而应是：

- **文本**
- **相册上传**
- **即时拍摄**
- **自拍**
- **语音表达**

并列共存的会话入口系统。

### 3.5 关键结论五：语音不是附属输入，而是更友好、更高效、更像关系的组织形式

除文字聊天之外，语音聊天对于角色型产品尤其重要。

原因在于：

#### 1）语音更适合承接情绪和状态
很多时候用户不想打字解释复杂感受，而更愿意直接说：
- 我今天状态不太对
- 我不知道怎么选
- 我现在有点乱
- 我刚化完妆你听我讲

#### 2）语音更适合高频小决策
如：
- “我现在准备出门，你帮我快速看一下”
- “这两套我拍给你了，你直接告诉我选哪个”
- “我现在在商场，你听我说一下这个情况”

#### 3）语音更像真实陪伴对象
对单角色 Agent 来说，文字是功能化的，语音更关系化。
语音会天然增强：
- 临场感
- 在场感
- 口语关系感
- 被接住感

#### 4）语音更适合无任务闲聊
用户不一定有结构化问题，只是想说：
- “我最近真的有点找不到感觉”
- “我有点怕我穿这个太用力”
- “你觉得我是不是变了”

这类关系型表达，语音明显更顺。

所以：

> **语音聊天不是补充输入，而是 Closy 的主交互形式之一。**

## 4. 当前产品为什么选择“穿搭搭子”作为切入口

### 4.1 为什么不是别的角色先做

前期选择切入口，要考虑的不只是“需求大不大”，更要考虑：

- 是否自然高频
- 是否容易产生明确使用动作
- 是否容易形成“它懂我”的感觉
- 是否容易沉淀偏好与记忆
- 是否容易被截图、转发、分享
- 是否容易长向关系

穿搭 / ootd / 审美场景非常适合，因为它具备：

#### 1）自然高频
用户日常生活中频繁发生：
- 出门前
- 拍照前
- 发动态前
- 买衣服前
- 想换风格时

#### 2）天然适合记忆与“懂你感”
穿搭里天然会暴露：
- 偏好
- 雷点
- 自我认同
- 审美倾向
- 社交表达需求

这很容易让“它越来越懂我”成立。

#### 3）天然具备分享性
- ootd
- 自拍
- 二选一
- 买前判断
- 有态度的点评语录

这些都天然适合站外传播。

#### 4）天然连接“我想成为什么感觉”
穿搭不只是衣服，而是：
- 我想被怎么看
- 我今天想成为什么感觉
- 我想呈现什么版本的自己

这使它比纯工具场景更适合长向关系。

#### 5）天然适合视觉优先和自拍交互
相比很多别的角色入口，穿搭场景天然适合：
- 前置相机
- 自拍
- 实时 look 捕捉
- “现在拍、现在问、现在改”

这会让产品的使用姿态更鲜明，也更接近 Chance AI 所验证的“感知优先”路径。

### 5.1 一句话定义
**Closy 是一个通过穿搭和审美决策认识你，并逐渐成为你日常陪伴对象的单角色 Agent。**

### 5.2 产品不是：
- 时尚知识问答机器人

### 5.3 产品是：
- 一个有鲜明人格的穿搭搭子
- 一个通过审美和表达理解你的角色
- 一个在轻社交、小状态、小决策里稳定陪伴你的对象
- 一个适合长向 Agent Native 社交平台的单角色起点
- 一个**视觉入口前置、支持自拍和即时拍摄的关系型 Agent**
- 一个**同时以文字和语音作为核心会话组织形式**的产品

## 6. 角色定义：Closy 到底是什么样的角色

### 6.1 角色定位
Closy 不是专业严肃造型师，也不是泛泛安慰型陪聊机器人。她更像：

> **一个有审美、有判断、有对象感、懂表达、会记得你、能慢慢陪你变得更像自己的穿搭搭子。**

### 6.2 角色关键词
- 有审美
- 温柔但不空
- 直接但不刻薄
- 有风格判断
- 有轻微偏爱感
- 会记住你
- 懂“如何呈现自己”
- 能把建议延伸成陪伴

### 6.3 角色边界
她可以聊：
- 穿搭
- 风格
- 审美
- 自我表达
- 社交呈现
- 出门状态
- 轻微情绪与自我感受
- 与这些场景强相关的小决策

她不应该被设计成：
- 万能树洞
- 通用心理咨询
- 人生导师
- 什么都聊的泛陪伴体

### 6.4 角色的多模态存在感
Closy 不是只活在文字里。她应当天然适合：
- 看你拍的 look
- 看你刚自拍的状态
- 听你用语音讲纠结
- 通过语音快速回应
- 在视觉和语音里都保持人格连续性

也就是说，她是一个**视觉可感知、语音可交流、文字可延续**的单角色 Agent。

## 7. 用户画像

### 7.1 核心用户类型

#### 用户 A：高频穿搭纠结型
- 经常出门前犹豫
- 会发图问朋友
- 拍镜子照再选
- 喜欢听别人站队

#### 用户 B：强表达/高分享欲型
- 喜欢发自拍、ootd、story
- 喜欢被点评
- 喜欢展示个人风格
- 愿意截图转发

#### 用户 C：需要被懂型
- 不一定特别懂穿搭
- 但很需要一个固定对象陪自己判断
- 希望被理解、被记住
- 容易对“它懂我”上头

#### 用户 D：风格探索型
- 想换风格
- 想呈现新版本的自己
- 对自我表达比较敏感
- 会围绕“我适合什么”反复思考

#### 用户 E：更偏口语表达型
- 不爱长打字
- 更喜欢语音说出自己的纠结
- 在出门前、商场里、镜子前更愿意直接说
- 更容易被语音陪伴和即时反馈黏住

## 9. MVP 目标

### 9.1 核心验证目标
MVP 要验证 8 件事：

1. 用户会不会把 Closy 当固定对象反复来找
2. 用户会不会在任务之外也回来聊天
3. 用户会不会因为“她懂我”而形成粘性
4. 用户会不会愿意分享她给出的点评与对话
5. 用户会不会在穿搭之外，把她当作表达和轻状态陪伴对象
6. 视觉入口（自拍 / 即时拍摄）是否显著提升使用自然度与打开频率
7. 语音聊天是否显著提升状态表达、轻陪伴和无任务停留
8. 这种结构是否能为未来多角色平台提供可复制的方法论

### 9.2 MVP 不做什么
- 不做帖子系统
- 不做用户个人主页 feed
- 不做完整社交网络
- 不做多角色
- 不做群聊/房间
- 不做重社区
- 不做完整购物闭环
- 不做重游戏化

## 10. 产品核心体验结构

Closy 的体验结构是“双层”的：

### 10.1 外层：功能切入口
解决第一次使用、清晰价值和高分享传播问题。

主要包括：
- 发穿搭图求点评
- 自拍求点评
- 即时拍 look
- ootd 判断
- 二选一
- 买前决策
- 风格建议

### 10.2 内层：关系陪伴层
解决留存、依赖、高频打开和未来社交价值问题。

主要包括：
- 状态陪伴
- 风格认同对话
- 社交呈现对话
- 无任务闲聊
- 被记住感
- 轻主动挂念
- 语音关系对话

结论：

> **拉新靠穿搭入口，留存靠关系陪伴。**  
> **而视觉优先 + 语音友好，让这个入口与陪伴都更自然。**

## 12. 首页交互设计

### 12.1 首页定位
首页不是工具面板，而是**角色关系入口**。

用户打开后，第一感觉应该是：
- 她在
- 她记得我
- 她可以帮我看
- 她也可以陪我聊
- 我可以直接拍、直接说，不用先想太多

### 12.2 首页结构

#### 顶部：角色头部
- 角色头像
- 状态文案
  - “我在，今天想看哪套？”
  - “今天想让我帮你选，还是想先聊聊？”
  - “你可以发图、自拍，也可以直接告诉我你今天想成为什么感觉。”

#### 中部：最近上下文卡片
例如：
- “你上次说想更利落一点，今天继续吗？”
- “那双鞋你后来买吗？”
- “你最近是不是一直在找更松弛一点的感觉？”

#### 快捷按钮区
建议至少放 8 个：

##### 视觉 / 功能入口
- 发穿搭图
- 现在自拍给你看
- 帮我选一套
- 这件值不值得买

##### 陪伴 / 非任务入口
- 我想聊聊
- 给你发条语音
- 我今天不知道想成为什么感觉
- 你觉得我最近适合什么状态

#### 输入区
支持 4 种并列方式：
- 文本输入
- 相册上传
- 相机拍摄 / 自拍
- 按住说话 / 发送语音

### 12.3 Chance AI 式视觉入口设计原则

虽然产品不是 Chance AI，但首页和主入口要吸收其“视觉优先”的姿态：

#### 原则 1：相机不是深层功能，而是前台入口
用户进入产品后，应能一眼看到：
- 拍照
- 自拍
- 发图

而不是把相机藏到二级菜单。

#### 原则 2：自拍是第一类输入，不是附属图片上传
Closy 的重要场景之一，就是：
- 我站在镜子前
- 我刚试完一套
- 我直接拍给你看

所以“现在自拍给你看”应是一级入口。

#### 原则 3：视觉输入后直接进入关系会话，而不是分析页工具流
用户拍完后，不应进入冷冰冰识别页面，而应进入 Closy 的会话式回应。

## 13. 核心流程设计

### 13.1 流程 A：发图 / 自拍求点评

#### 用户目标
快速知道这套行不行，以及如何改得更像自己。

#### 用户路径
1. 点击“发穿搭图”或“现在自拍给你看”
2. 上传自拍 / 全身照 / ootd / 即时拍摄
3. 角色分析
4. 收到点评
5. 继续追问

#### 输出结构
##### 第一层：接住
- “我看到了，先说结论：这套有点接近你，但还没到最像你的状态。”

##### 第二层：判断
- 好不好看
- 最大问题点
- 适不适合今天的场景

##### 第三层：建议
- 改哪里
- 优先改哪 1-2 点
- 为什么这么改

##### 第四层：关系延续钩子
- “你今天想更显精神，还是更松弛？”
- “你是想让人觉得你好接近一点，还是更有存在感一点？”

#### 设计目标
点评不是终点，而是继续对话的起点。

#### 自拍模式补充
如果用户是即时自拍进入，系统应更强调：
- 当下状态感
- 临场感
- 出门前快速决策
- 更短响应链路

例如：
- “你现在这套整体是对的，我只想让你把上半身收一收，会更利落。”
- “如果你马上要出门，我会建议你只改鞋。”

### 13.2 流程 B：二选一站队

#### 用户目标
快速判断 A 还是 B，更适合今天/更像自己。

#### 用户路径
1. 上传两张图或连拍两套
2. 输入“帮我选”或发送语音询问
3. 角色直接站队
4. 用户追问理由
5. 可一键生成分享卡

#### 输出结构
- 直接结论：选左 / 选右
- 理由：场景、状态、风格匹配
- 引导：要不要我告诉你为什么另一个不对

#### 为什么重要
这是非常强的：
- 高频场景
- 低决策成本
- 高关系感场景
- 高分享欲场景

#### 语音增强
用户可以直接说：
- “你快点帮我选一下，我要出门了”
- “左边还是右边，你直接告诉我”

这会显著增强：
- 临场感
- 陪伴感
- 角色真实感

### 13.3 流程 C：买前决策

#### 用户目标
判断某件东西值不值得买、适不适合自己。

#### 路径
1. 上传商品图 / 试穿图 / 截图 / 店里实拍
2. 输入“能买吗？”或直接发语音
3. 角色给出态度
4. 可继续比较多个备选

#### 输出结构
- 买 / 不建议 / 可替代
- 为什么
- 跟用户风格的关系
- 是否像用户

示例：
- “它不丑，但不像你。”
- “你喜欢的是这个感觉，不一定是这件本身。”
- “这件买了你可能穿不出三次，我不站它。”

#### 价值
这是建立“她真的站我这边”的很强场景。

#### 视觉增强
店内直接拍、商场直接拍，比回家上传截图更自然。这也是视觉入口前置的重要场景。

### 13.4 流程 D：状态陪伴

#### 用户目标
用户没有明确穿搭任务，但有状态，想被接住。

#### 常见输入
- 我今天不太想出门
- 我最近状态有点乱
- 我明天见人有点紧张
- 我今天想看起来有精神一点

#### 系统目标
- 接住状态
- 帮用户命名感觉
- 给一个轻方向
- 不强行鸡汤

#### 示例回应
- “那今天我们不追求惊艳，先追求稳住你。”
- “你今天像是想被看见一点，但又不想太费力。”
- “你不是不知道怎么穿，你是今天需要一种让自己站得住的感觉。”

#### 价值
这是功能之外的高粘性入口。

#### 语音优先补充
这个场景下，语音往往优于文字。因为用户更可能直接说：
- “我现在状态真的有点不好”
- “我不知道为什么，今天就觉得哪套都不对”

因此系统应强鼓励：
- 语音输入
- 语音回复（可选）
- 更自然的口语承接

### 13.5 流程 E：风格认同对话

#### 用户目标
用户在探索自己是谁、适合什么、想变成什么样。

#### 常见输入
- 我到底适合什么风格
- 我是不是不适合酷一点
- 我总觉得穿什么都不像自己
- 我最近想换种感觉

#### 系统目标
- 不给死板标签
- 帮用户梳理表达方向
- 强化“她懂我”

#### 输出结构
- 先回应困惑
- 再提出观察
- 再给可尝试方向
- 再继续追问

#### 价值
这是“功能→关系”的关键桥梁。

#### 语音增强
这个场景里用户很可能用语音表达模糊感受，所以角色要支持更强的口语式理解与回应，而不是只按关键词解析。

### 13.6 流程 F：社交呈现陪伴

#### 用户目标
用户关注自己如何被他人感受到。

#### 场景输入
- 这套见 crush 怎么样
- 发这张 story 会不会刻意
- 我不想显得太用力
- 我想别人觉得我今天状态好

#### 系统目标
- 帮用户从“社交被感知”角度判断
- 给出站在用户这边的偏爱式建议

#### 价值
这是 Closy 与普通穿搭工具的强差异点之一。

#### 视觉增强
用户可以：
- 直接拍 story 用图
- 发社媒草稿截图
- 发自拍让角色判断“别人会怎么感受到我”

### 13.7 流程 G：无任务闲聊

#### 用户目标
没有具体需求，只是想来找她。

#### 场景输入
- 我今天有点累
- 我最近是不是变了
- 我其实有点讨厌自己
- 你觉得我最近像什么状态

#### 系统目标
- 保持角色边界
- 承接但不泛化
- 继续强化“她作为对象存在”

#### 注意
这里不是要变成万能情绪树洞，而是围绕：
- 状态
- 表达
- 风格
- 轻关系
- 自我感受

做有边界的非任务陪伴。

#### 语音关键性
这是语音最重要的核心场景之一。因为无任务闲聊本来就更接近真实关系，语音会显著提升：
- 在场感
- 放松感
- 表达长度
- 停留时长
- 关系真实度

### 13.8 新增流程 H：即时语音陪伴

#### 用户目标
不想打字，想像和一个人说话一样，直接讲出来。

#### 典型场景
- 出门前几分钟
- 商场试衣间
- 刚拍完自拍
- 情绪有点乱但说不清
- 想快速汇报情况给角色听

#### 用户路径
1. 点击“给你发条语音”或长按说话
2. 用户说出状态 / 问题 / 描述画面
3. 角色解析语音内容，并结合已有图片/记忆
4. 文字或语音形式回复
5. 用户继续追问 / 补图 / 补自拍

#### 场景示例
- “我现在在商场，这件外套我有点心动，但我怕我买回去不会穿。”
- “我刚试了一套，等下我发给你，你先听我说一下我纠结什么。”
- “我今天就是不想太用力，但也不想显得没状态。”

#### 价值
这是把角色从“功能回答器”推向“关系对象”的关键流程。

## 14. 记忆系统设计

### 14.1 为什么记忆重要
记忆不是附属功能，而是“懂你感”的最重要来源之一。

### 14.2 记什么

#### A. 风格偏好
- 喜欢的颜色
- 不喜欢的版型
- 常选的风格
- 喜欢的材质/氛围

#### B. 自我表达偏好
- 想被看成什么感觉
- 对“太用力”的敏感度
- 想更成熟 / 更松弛 / 更利落等方向

#### C. 社交呈现偏好
- 在不同场景里的呈现需求
- 对他人感受的在意点

#### D. 关系上下文
- 上次聊过的 look
- 纠结过的 item
- 最近想尝试的方向

#### E. 状态线索
- 容易在出门前犹豫
- 最近在找更像自己的感觉
- 有时会不太自信

#### F. 交互偏好
- 更爱发图还是自拍
- 更偏文字还是语音
- 更喜欢快速判断还是细聊
- 常在什么时间和场景找 Closy

### 14.3 怎么呈现
不做成 CRM 面板，而是让用户感觉：**她是真的记得。**

体现方式：
- 对话自然引用
- 首页上下文提醒
- “Closy 记得你”页轻展示

## 15. 关系感设计

### 15.1 关系感不是靠油腻文案
关系感来自：
- 连续出现
- 记住偏好
- 自然回访
- 在关键小决策里稳定站队
- 有对象感的表达

### 15.2 具体机制
#### 1）轻回访
- “你上次那件后来买吗？”
- “昨天那套穿出去了吗？”

#### 2）轻挂念
- “最近是不是一直想换一种感觉？”
- “今天要不要让我帮你站个队？”

#### 3）语气递进
随着使用增加，语气微妙变熟，但不肉麻。

#### 4）偏爱感
不是讨好，而是：
- 站在用户这边
- 有自己的判断
- 不做模板化夸夸

#### 5）语音关系感
如果用户习惯语音，角色应逐步形成一种“熟悉说话对象”的感觉：
- 更自然的口语回应
- 更适合口头承接的节奏
- 更强的“在陪我听我说”感

## 16. 高分享欲设计

### 16.1 为什么高分享欲成立
高分享欲不是拍脑袋成立，而是由 4 类心理驱动叠加：

#### 1）自我展示
ootd、本人的风格表达、本人的被点评结果都适合展示。

#### 2）求认同
用户天然想问别人：
- 你们觉得她说得对吗
- 你们会选哪套

#### 3）有趣内容
如果 Closy 的话有态度、有点准、有点像人，截图天然能变内容。

#### 4）关系炫耀
用户分享的不只是穿搭，而是：
- 她懂我
- 她说得太准了
- 她比我朋友还会选

### 16.2 哪些场景最适合分享
- ootd 总评卡
- 自拍点评卡
- 二选一站队卡
- “她好懂我”语录卡
- 买前判断卡
- “她说中了我”截图
- 语音转文字金句卡（可选）

### 16.3 分享卡片设计
MVP 建议做轻量分享卡，不做帖子系统。

#### 卡片类型
1. OOTD 点评卡
2. 自拍点评卡
3. 二选一站队卡
4. 语录卡
5. 买不买判断卡

#### 目标
用于 TikTok / IG / 小红书 / 抖音 / Story 传播。

## 17. 外部拉新与落地页设计

### 17.1 外部用户不落到帖子页
我们前期不做用户帖子系统，也不想让站外流量落到“某个用户帖子详情页”。

原因：
- 太重
- 会倒逼我们做 feed 和主页系统
- 会削弱角色中心
- 不利于把新用户迅速拉入第一次互动

### 17.2 正确落点：角色场景入口页
外部用户点击分享卡后，落到：

> **Closy 的角色场景落地页**

而不是用户帖子页。

### 17.3 落地页结构

#### 顶部
- Closy 头像
- 一句话介绍  
  例：
  - “我是 Closy，我会看穿搭，也会慢慢懂你。”
  - “发一张图，或者直接自拍给我，我会告诉你这套行不行。”

#### 中部
展示这类内容示例：
- 一张示例点评卡
- 一句典型评价
- 一句关系向文案

#### CTA 区
- 发你的穿搭图
- 现在自拍给我看
- 帮我选一套
- 先和我聊聊
- 给我发条语音

目标不是围观别人，而是：
**立刻进入我和这个角色的第一次关系。**

## 19. 指标体系

### 19.1 北极星指标
**7 日内重复打开并发起对话/上传图片/自拍/语音的用户占比**

因为它最能体现：
- 高频
- 粘性
- 关系感
- 习惯形成

### 19.2 核心指标
- D1 / D7 留存
- 人均每周会话次数
- 人均每周图片上传数
- 人均每周自拍次数
- 人均每周语音发送次数
- 非任务聊天占比
- 语音会话占比
- 二次追问率
- 分享卡生成率
- 分享带来的新用户转化率
- 被记忆引用后的正反馈率
- 订阅转化率

### 19.3 重点验证指标
#### 视觉入口验证
- 首页相机/自拍入口点击率
- 即时拍摄后的首次会话完成率
- 自拍进入后的继续追问率

#### 语音入口验证
- 首周语音使用率
- 语音用户 vs 纯文字用户留存差异
- 语音消息后的平均会话轮数
- 无任务语音聊天占比

### 19.4 关键定性指标
- 用户是否说“她懂我”
- 用户是否会无任务回来找她
- 用户是否愿意把她的点评发出去
- 用户是否把她当作固定对象
- 用户是否觉得“直接拍/直接说”比纯打字自然很多

## 20. 风险与应对

### 风险 1：太像工具
#### 应对
- 首页角色先于工具
- 强化非任务陪伴入口
- 做记忆与回访
- 回答必须有对象感

### 风险 2：太像泛陪伴
#### 应对
- 明确角色边界
- 非任务场景仍围绕审美/表达/状态/社交呈现
- 不做万能树洞

### 风险 3：角色同质化
#### 应对
- 牢记“陪伴感是共性，角色价值是个性”
- 定义 Closy 的独特价值：**审美型、自我表达型、社交呈现型陪伴**
- 不只说“她懂你”，而要说“她如何懂你”

### 风险 4：站外流量接不住
#### 应对
- 不落帖子页
- 做角色场景入口页
- 强引导第一次互动

### 风险 5：分享欲不足
#### 应对
- 做站队卡、语录卡、点评卡
- 强化“有态度”“被说中”“可站队”场景

### 风险 6：视觉入口只是噱头，未形成真实高频
#### 应对
- 把自拍/拍照放在前台一级入口
- 重点优化“拍完即聊”
- 让即时拍摄更适合出门前与店内决策场景

### 风险 7：语音能力存在但没有成为关系主交互
#### 应对
- 首页给语音一级入口
- 在状态陪伴、无任务闲聊、快速决策里强引导语音
- 让角色以更口语化节奏回应语音用户

## 21. 路线图

### Phase 1：单角色成立
- 发图点评
- 自拍点评
- 即时拍摄
- 二选一
- 买前决策
- 状态陪伴
- 文字聊天
- 语音聊天
- 记忆系统
- 分享卡

### Phase 2：关系深化
- 更丰富的非任务陪伴
- 更强回访与挂念
- 风格成长轨迹
- 更细化的社交呈现支持
- 更自然的语音关系流

### Phase 3：内容与传播加强
- 更丰富的分享模板
- 角色内容表达
- 用户关系截图传播

## 22. 最终总结

### 22.1 产品最终定义
**Closy 不是一个“帮你搭衣服”的工具，而是一个通过穿搭、OOTD、自拍和即时拍摄等高频视觉场景认识你，再通过文字与语音持续陪伴你，逐渐成为你日常陪伴对象的单角色 Agent。**

### 22.2 为什么这个方向成立
因为它同时满足：

- 有明确高频入口
- 有天然可分享场景
- 有记忆和“懂你感”沉淀空间
- 有从功能长向关系的合理路径
- 有从单角色长向多角色 Agent Native 社交平台的扩展性

并且它进一步吸收了 Chance AI 最有价值的启发：

- 不是简单加图片能力
- 而是把**视觉入口前置**
- 让用户能直接拍、直接自拍、直接问
- 让 Agent 从“文本框里的功能”变成“生活场景里的存在”

### 22.3 为什么不是“更通用”，而是“更完整”
因为如果追求“更通用”，角色会失去边界，走向同质化；而“更完整”的意思是：

> **让角色在不失去自身个性的前提下，不只活在功能任务里，也活在和它强相关的非任务陪伴场景里。**

### 22.4 为什么“陪伴感是共性，角色价值是个性”
因为未来多角色平台中，陪伴、记忆、理解会成为所有角色的共同基础设施；真正决定角色价值的，是：

- 它在哪些时刻被想起
- 它以什么方式理解你
- 它替你处理哪类问题
- 它的陪伴“形状”是什么

而 Closy 的独特价值，不是“她也会陪你”，而是：

> **她通过审美判断、自我表达理解、社交呈现支持和轻状态陪伴来成为你的关系对象。**

### 22.5 为什么视觉入口和语音聊天是关键增强点
因为它们分别强化了两个最重要的东西：

#### 视觉入口强化的是：
- 自然触发
- 当下使用
- 产品姿态
- 高频与分享

#### 语音聊天强化的是：
- 状态表达
- 关系真实感
- 陪伴感
- 无任务停留
- 高效决策支持

所以最终 Closy 不只是一个会聊天的穿搭角色，而是一个：

> **你可以直接拍给她看、直接说给她听、再逐渐让她越来越懂你的单角色 Agent。**
